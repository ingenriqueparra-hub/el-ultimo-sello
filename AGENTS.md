# Guía secundaria para Codex en **El Último Sello**

`CLAUDE.md` es el **punto de entrada principal** del proyecto para agentes IA.

Este archivo existe para compatibilidad con Codex cuando el entorno espera un `AGENTS.md`. Antes de diseñar, programar o modificar cualquier parte del juego, Codex debe leer y obedecer primero `CLAUDE.md`.

Si hay conflicto entre `CLAUDE.md` y este archivo, priorizar `CLAUDE.md`.

## Regla operativa para Codex

1. Leer `CLAUDE.md` como guía principal.
2. Consultar los documentos indicados por `CLAUDE.md` según el tipo de tarea.
3. Mantener el alcance del MVP.
4. No introducir sistemas grandes fuera de `MVP_BACKLOG.md` o `TECHNICAL_SPEC.md`.
5. Responder en español, con tono ejecutivo, claro y útil.
6. Usar Godot 4 y GDScript para código, salvo instrucción contraria.
7. Priorizar universo propio y bajo riesgo legal.

## Archivos principales

- `CLAUDE.md`: guía principal para agentes IA.
- `PROJECT.md`: visión, alcance, MVP y estado del proyecto.
- `TECHNICAL_SPEC.md`: arquitectura Godot 4 y estructura técnica.
- `MVP_BACKLOG.md`: alcance cerrado y prioridades del MVP.
- `GAME_DESIGN.md`: mecánicas, reglas y loop jugable.
- `IP_SAFETY.md`: límites legales y protección de IP propia.

Codex no debe duplicar ni reinterpretar las reglas principales aquí. Este archivo solo apunta al documento correcto.
